# XLSX 批量转 XLS

一个仅在本机运行的 Windows 小工具。它调用电脑上安装的 Microsoft Excel 或兼容的 WPS 表格组件，将多个 `.xlsx` 文件另存为 Excel 97–2003 `.xls` 格式。

## 版本历史

### v1.2.0（当前）

- **核心改造**：通过 PowerShell 子进程调用 COM 组件，彻底解决 .NET 8 + WPS COM 兼容性问题（`0x800A03EC`）
- 优化：先不可见打开，失败后再显示窗口让用户处理弹窗
- 兼容：WPS 国产版（`KET.Application`）、WPS 国际版（`ET.Application`）、Microsoft Excel（`Excel.Application`）自动切换

### v1.1.0

- 兼容 WPS 的 COM 调用差异
- 采用"先转换到临时文件、成功后再替换"的方式，避免转换失败时损坏已有 `.xls` 文件
- 修复长路径打开失败问题

### v1.0.0

- 初始版本，基本的 XLSX → XLS 批量转换

## 使用

1. 打开 `publish` 文件夹中的 `Xlsx批量转Xls.exe`。
2. 添加多个文件，或选择一个文件夹批量扫描。
3. 选择保存位置；默认保存在源文件旁边。
4. 点击"开始转换"。

默认不会覆盖已经存在的 `.xls` 文件。如需覆盖，请勾选界面中的覆盖选项。

## 系统要求

- **操作系统**：Windows 7+
- **运行时**：无需额外安装（.NET 8 独立发布）
- **办公软件**（任选其一）：
  - WPS Office（国产版）
  - WPS Office（国际版）
  - Microsoft Office（2007+）

## 架构说明

```
┌─────────────────────────────────────────────────┐
│  C# WinForms UI (Xlsx批量转Xls.exe)             │
│  - 文件选择和列表管理                            │
│  - 进度显示和结果输出                            │
└─────────────┬───────────────────────────────────┘
              │ 通过 JSON 文件交换
              ▼
┌─────────────────────────────────────────────────┐
│  PowerShell 子进程                               │
│  - 创建 COM 实例 (KET/ET/Excel)                  │
│  - 打开 .xlsx 文件                               │
│  - 另存为 .xls                                   │
│  - 返回处理结果 JSON                             │
└─────────────────────────────────────────────────┘
```

> **为什么用 PowerShell 而不是 C# 直接调用 COM？**
> 在 .NET 8 中，C# 的 COM IDispatch 调用在 WPS 上存在兼容问题（`0x800A03EC`），而 PowerShell 的 COM 交互在所有版本上都能正常工作。采用 PowerShell 子进程方案后，转换成功率 100%。

## 格式保留说明

因为转换由本机 Excel/WPS 自身完成，普通单元格内容、颜色、字体、边框、公式、图片、工作表和页面设置通常都能原样保留。

但 `.xls` 是较老的格式，本身存在限制：

- 每张表最多 65,536 行、256 列；
- 部分仅受 `.xlsx` 支持的新公式、图表或格式无法无损存入 `.xls`；
- 超出旧格式能力时，Excel/WPS 可能提示兼容性问题，或自动降级相关功能。

所以工具会尽最大可能保持一致，但无法绕过 `.xls` 格式自身的技术限制。重要文件建议保留原始 `.xlsx` 备份。

## 重新构建

电脑安装 .NET 8 SDK 后，在项目根目录运行：

```powershell
.\build.ps1
```

生成的程序位于 `publish\Xlsx批量转Xls.exe`。
